# Shared React components

## Why this folder exists

Presentational and composite UI reused across routes. No domain stores here — use features/.

## What is here

- ui/ — primitives (buttons, dialogs)
- home/, site/ — marketing sections
- products/ — catalog cards
- shared/ — cross-page widgets

## Ownership

| Area | Canonical owner | Typical consumers | Do not duplicate in |
|------|-----------------|-------------------|---------------------|
| Marketing sections & chrome | `components/home/, components/site/, components/products/` | app/(site)/ routes | features/planner/ — planner workspace UI |
| App-wide UI primitives | `components/ui/` | components/*, features/shared, app/ | features/planner/shared/components/ — planner-only chrome |
| Cross-route widgets (site) | `components/shared/` | marketing routes | features/shared/components/ — auth badges, shell pieces |
| Auth & shell (stateful) | `features/shared/components/, features/shared/auth/components/` | login, access, dashboard layouts | components/ — keep dumb/presentational |

## Rules

- Prefer features/ for stateful domain UI
- ui/ stays dumb and styled via FOCSS tokens
