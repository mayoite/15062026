# Canonical planner

## Why this folder exists

Single source for /planner workspace: Fabric editor, catalog placement, 3D viewer, persistence, AI assist.

## What is here

- editor/ — canvas, layers, tools, inspector
- catalog/ — ingest + generated items
- store/ — Zustand state
- 3d/, viewer/ — Three.js scene
- landing/, onboarding/ — guest funnel
- persistence/, document/, model/
- shared/ — BOQ, export, engine types
- canvas-fabric/ — Fabric canvas shapes & tools

## Ownership

| Area | Canonical owner | Typical consumers | Do not duplicate in |
|------|-----------------|-------------------|---------------------|
| Canvas & workspace shell | `features/planner/editor/, features/planner/ui/` | app/planner/(workspace)/ | app/planner/ — routes only, no editor logic |
| Planner-specific React chrome | `features/planner/components/, features/planner/shared/components/` | features/planner/editor, features/planner/landing | components/ — marketing UI only |
| Client state (Zustand) | `features/planner/store/` | editor, hooks, persistence | lib/store/ — site quote cart only |
| Document schema & permissions | `features/planner/model/` | persistence, API /api/plans | platform/drizzle/ — DB row shape only |
| Site product catalog | `features/catalog/ + lib/catalog/` | features/planner/catalog via catalogBridge | Duplicate ingest in planner/catalog/ except placement adapters |

## Dependencies

- `fabric` — 2D canvas rendering engine
- `three` + `@react-three/fiber` + `@react-three/drei` — 3D rendering
- `zustand` — State management
- `@/features/planner/catalog/` — Product catalog integration
- `@/platform/supabase/` — Data persistence
- `@/features/shared/auth/` — Authentication

## Architecture Notes

- Zustand drives all state → Fabric renders 2D → document model feeds 3D viewer
- Offline-first: IndexedDB stores plans locally, sync queue pushes to Supabase when online
- 2D and 3D are decoupled — 3D reads from document model, not Fabric directly
- Autosave runs on every mutation with debouncing
