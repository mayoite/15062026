# Cross-feature shared code

## Why this folder exists

Auth, shell layout, analytics hooks used by site, planner, and admin.

## What is here

- `auth/lib/AuthProvider.tsx` — React context provider for Supabase auth state
- `auth/lib/humanizeAuthError.ts` — User-friendly auth error message formatting
- `auth/components/LoginPage.tsx` — Reusable login page component
- `auth/components/SignupPage.tsx` — Reusable signup page component
- `auth/components/ResendVerificationButton.tsx` — Email verification resend
- `auth/types.ts` — Shared auth types (SharedSessionUser, PlannerRole, SessionState)
- `dashboard/DashboardClient.tsx` — Authenticated user dashboard
- `analytics/types.ts` — Analytics event type definitions

## Dependencies

- `@supabase/supabase-js` — Auth client
- `@supabase/ssr` — SSR-compatible session management
- `@/platform/supabase/` — Supabase client configuration
- `@/lib/auth/` — Server-side auth helpers

## Used By

- `app/(site)/` — Site-wide auth provider wrapping
- `app/planner/` — Planner authentication
- `app/admin/` — Admin authentication
- `features/planner/` — Planner session management
- All protected routes via `lib/auth/session.ts`

## Security Notes

- Supabase auth state managed via SSR-compatible cookie handling
- Session refresh handled automatically by `@supabase/ssr`
- Role-based access: owner, editor, viewer, guest
- Guest access allowed on specific planner routes only
