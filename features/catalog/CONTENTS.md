# Catalog domain

## Why this folder exists

Product filtering, schemas, and resolvers shared by site catalog and planner.

## What is here

- `categories.ts` — Category tree definitions, labels, descriptions, ID normalization
- `getProducts.ts` — Product fetching from Supabase, normalization to CompatProduct type
- `imageMetadata.ts` — Image asset metadata, resolution, CDN path handling

## Dependencies

- `@/platform/supabase/` — Product data queries
- `@/platform/drizzle/` — ORM for catalog tables
- `@/lib/assetPaths.ts` — Asset path normalization
- `@/lib/productDataTables.ts` — Product specs and images data tables

## Used By

- `app/(site)/products/` — Public product listing and detail pages
- `features/planner/catalog/` — Planner furniture placement
- `app/api/recommendations/` — AI recommendation engine
- `app/api/admin/catalog/` — Admin catalog management

## Architecture Notes

- Hierarchical category structure: top-level categories with sub-categories
- Product data split across multiple Supabase tables (products, product_specs, product_images)
- `CompatProduct` type bridges legacy and new catalog formats
- Image paths normalized for CDN (DigitalOcean Spaces → Cloudflare)
